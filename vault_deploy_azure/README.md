# vault_deploy_azure

Deploys a single-node **Vault Enterprise** cluster on an Azure Linux VM using Docker and Azure Key Vault for auto-unseal and secret storage. Equivalent to `vault_deploy_aws` but targeting Azure.

## Architecture

```
Azure Resource Group
└── Virtual Network (module-managed or BYOVNET)
    └── Subnet
        └── Network Security Group
            ├── Allow TCP 8200 (Vault API/UI)
            └── Allow TCP 22   (SSH — optional)
        └── Public IP (Static, Standard SKU)
        └── Network Interface
        └── Linux VM (Ubuntu 22.04 LTS)
            ├── custom_data → cloud-init bootstrap script
            └── User-Assigned Managed Identity
Azure Key Vault (Premium SKU)
├── RSA key  → Vault auto-unseal (wrap/unwrap)
└── Secrets  → root token + 5 recovery keys (written by cloud-init)
```

| AWS Component | Azure Equivalent |
|---|---|
| KMS Customer Managed Key | Azure Key Vault RSA key |
| SSM Parameter Store (SecureString) | Azure Key Vault Secret |
| IAM Instance Profile | User-Assigned Managed Identity |
| `seal "awskms"` | `seal "azurekeyvault"` |
| AWS IMDS | Azure IMDS (`169.254.169.254/metadata/...`) |

## Prerequisites

- Terraform ≥ 1.5.0
- `az login` — authenticated Azure CLI session (or service principal via environment variables)
- An existing **Azure Resource Group**
- An SSH public key string

## Quick Start

```bash
cd examples/azure
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars — set resource_group_name, admin_ssh_public_key, etc.
export TF_VAR_vault_license="<your license>"
terraform init
terraform apply
```

After apply completes (~3–5 minutes for cloud-init):

```bash
# Retrieve the root token
az keyvault secret show \
  --vault-name $(terraform output -raw key_vault_name) \
  --name vault-root-token \
  --query value -o tsv

# Test connectivity (the cert is self-signed — use -k or supply VAULT_CACERT)
export VAULT_ADDR=$(terraform output -raw vault_addr)
export VAULT_TOKEN=<root_token>
vault status
```

## Networking — BYOVNET

To deploy into an existing VNet, supply both `vnet_id` and `subnet_id`:

```hcl
module "vault" {
  source = "../../vault_deploy_azure"
  ...
  vnet_id   = "/subscriptions/.../virtualNetworks/my-vnet"
  subnet_id = "/subscriptions/.../virtualNetworks/my-vnet/subnets/my-subnet"
}
```

## TLS

Vault uses a **self-signed certificate** generated by cloud-init. To avoid `-k` / `VAULT_SKIP_VERIFY`:

```bash
# Copy the cert from the VM
scp azureuser@<public_ip>:/opt/vault/certs/vault.crt ./vault.crt
export VAULT_CACERT=./vault.crt
```
